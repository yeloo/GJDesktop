## 二、当前实现说明

### 模块清单

1. **DesktopIconWriter**(图标写回器)
   - 职责:封装桌面图标位置写回逻辑
   - 主路线:IFolderView::SetItemPosition
   - 备用路线:SysListView32(v1 中标记为不可用)
   - 接口:moveIcons(), moveSingleIcon()

2. **DesktopArrangeRuleEngine**(规则引擎)
   - 职责:基于固定规则对图标进行分类
   - 分类:Folder, Shortcut, Image, Document, Archive, Executable, Other
   - 接口:classifyIcon(), getCategoryName(), getFixedCategories()

3. **DesktopLayoutPlanner**(布局规划器)
   - 职责:按固定区域规则计算每个图标的目标坐标
   - 布局:固定分区,简单网格排列,纵向优先
   - 接口:planLayout(), getLayoutRegion(), printLayoutPlan()

4. **DesktopAutoArrangeService**(自动整理服务)
   - 职责:提供统一的自动整理调用入口,串联全流程
   - 流程:读取 -> 身份构建 -> 分类 -> 布局规划 -> 写回
   - 接口:arrangeDesktop(), getIconAccessor(), getIconWriter(), getRuleEngine(), getLayoutPlanner()

### 数据流

```
DesktopIconSnapshot (readDesktopIcons)
  -> buildIconIdentity() -> ArrangeableDesktopIcon[]
  -> classifyIcon() -> ArrangeableDesktopIcon[] (带 category)
  -> planLayout() -> DesktopLayoutTarget[]
  -> moveIcons() -> AutoArrangeResult
```

### 关键设计决策

1. **身份标识**:使用 parsingName 作为主标识符(v1 简化实现中使用 displayName 代替)
2. **写回路线**:仅使用 COM 路线(ListView 路线 v1 中不可用)
3. **布局策略**:固定分区,简单网格,纵向优先,不做复杂配置化
4. **错误处理**:单个图标失败不影响整体,详细日志记录
5. **模块边界**:职责清晰,不堆砌逻辑
