# 真正自动整理桌面图标功能 - 第一轮实现汇报

## 一、实现策略

基于 Windows Shell COM 路线，实现策略如下：

1. **COM 路线优先**：通过 `IFolderView::SetItemPosition` 或 `IFolderView::SelectAndPositionItems` 写回图标位置
2. **利用 parsingName**：使用已有的 parsingName (SHGDN_FORPARSING) 作为稳定身份标识
3. **PIDL 定位**：通过 `ParseDisplayName` 将 parsingName 转换为 PIDL
4. **最小改动**：在现有 `DesktopIconWriter` 架构上扩展，不破坏已有逻辑

### 1. writeUsingCOMInterface() 实现核心逻辑

```cpp
bool DesktopIconWriter::writeUsingCOMInterface(
    const DesktopIconIdentity& identity,
    POINT targetPosition,
    std::string& errorMessage
) {
    // 1. 检查 parsingName 是否有效
    if (identity.parsingName.empty()) {
        errorMessage = "parsingName 为空，无法定位图标";
        return false;
    }

    // 2. 获取桌面 Shell Folder
    CComPtr<IShellFolder> pDesktopFolder;
    hr = SHGetDesktopFolder(&pDesktopFolder);
    if (FAILED(hr) || !pDesktopFolder) {
        errorMessage = "无法获取桌面 Shell Folder";
        return false;
    }

    // 3. 将 parsingName 转换为 PIDL
    // parsingName (UTF-8) -> wideParsingName (UTF-16)
    int wideLen = MultiByteToWideChar(CP_UTF8, 0, identity.parsingName.c_str(), -1, nullptr, 0);
    std::wstring wideParsingName(wideLen - 1, L'\0');
    MultiByteToWideChar(CP_UTF8, 0, identity.parsingName.c_str(), -1, &wideParsingName[0], wideLen);

    hr = pDesktopFolder->ParseDisplayName(
        nullptr, nullptr, &wideParsingName[0], &chEaten, &pidl, &dwAttributes
    );
    if (FAILED(hr) || !pidl) {
        errorMessage = "无法将 parsingName 转换为 PIDL";
        return false;
    }

    // 4. 获取桌面窗口的 IFolderView
    CComPtr<IShellWindows> pShellWindows;
    hr = CoCreateInstance(CLSID_ShellWindows, nullptr, CLSCTX_ALL, IID_PPV_ARGS(&pShellWindows));
    
    // 遍历所有窗口查找桌面
    for (long i = 0; i < hwndCount; i++) {
        // 查找 IShellBrowser -> IShellView -> IFolderView
        // ... (详细实现见源码)
    }

    // 5. 尝试设置图标位置
    // 方法 A: SetItemPosition
    hr = pFolderView->SetItemPosition(pidl, &pt);
    if (SUCCEEDED(hr)) {
        return true;
    }

    // 方法 B: SelectAndPositionItems (回退方案)
    hr = pFolderView->SelectAndPositionItems(1, pidlArray, ptArray, dwArray);
    if (SUCCEEDED(hr)) {
        return true;
    }

    return false;
}
```

### 2. moveSingleIcon() 调用 COM 路线

```cpp
bool DesktopIconWriter::moveSingleIcon(
    const DesktopIconIdentity& identity,
    POINT targetPosition,
    std::string& errorMessage
) {
    // 优先使用 COM 路线（IFolderView）
    bool success = writeUsingCOMInterface(identity, targetPosition, errorMessage);

    if (success) {
        Logger::getInstance().info("移动成功 - displayName: '%s', position: (%d, %d)",
            identity.displayName.c_str(), targetPosition.x, targetPosition.y);
        return true;
    } else {
        Logger::getInstance().error("移动失败 - displayName: '%s', error: %s",
            identity.displayName.c_str(), errorMessage.c_str());
        return false;
    }
}
```

## 附录：技术细节

### COM 接口使用

1. **IShellFolder**: Shell 文件夹接口，用于 ParseDisplayName
2. **IShellWindows**: Shell 窗口集合，用于查找桌面窗口
3. **IShellBrowser**: Shell 浏览器接口
4. **IShellView**: Shell 视图接口
5. **IFolderView**: 文件夹视图接口，用于 SetItemPosition

### PIDL 生命周期管理

使用 RAII 智能指针管理 PIDL：
```cpp
struct PIDLDeleter {
    void operator()(LPITEMIDLIST p) const { ILFree(p); }
};
std::unique_ptr<ITEMIDLIST, PIDLDeleter> pidlGuard(pidl);
```

int wideLen = MultiByteToWideChar(CP_UTF8, 0, identity.parsingName.c_str(), -1, nullptr, 0);
