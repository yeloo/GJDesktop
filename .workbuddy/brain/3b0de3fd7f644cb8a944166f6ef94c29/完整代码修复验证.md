## 二、关键修复点说明

### 1. logger.h 格式化实现修复

**问题**：原实现使用 `_snprintf(nullptr, 0, ...)` 在MSVC下不稳定，且存在std::string越界风险

**修复**：
```cpp
template<typename... Args>
static std::string formatMessage(const char* format, Args... args) {
    // 第一步：计算所需缓冲区大小
    int size = 0;
    {
        va_list va_args;
        va_start(va_args, format);
#ifdef _MSC_VER
        size = _vscprintf(format, va_args);
#else
        size = vsnprintf(nullptr, 0, format, va_args);
#endif
        va_end(va_args);
    }

    if (size <= 0) {
        return format;
    }

    // 第二步：分配足够大小的缓冲区
    std::vector<char> buffer(size + 1, '\0');

    // 第三步：执行实际的格式化
    {
        va_list va_args;
        va_start(va_args, format);
#ifdef _MSC_VER
        vsnprintf_s(buffer.data(), size + 1, _TRUNCATE, format, va_args);
#else
        vsnprintf(buffer.data(), size + 1, format, va_args);
#endif
        va_end(va_args);
    }

    // 第四步：转换为 std::string
    return std::string(buffer.data());
}
```

**关键改进**：
- 使用 `_vscprintf` (MSVC) 或 `vsnprintf(nullptr, 0)` 计算所需大小
- 使用 `vector<char>` 作为临时缓冲区，避免std::string越界
- `va_start(va_args, format)` - 正确的va_list初始化方式
- 使用 `vsnprintf_s` (MSVC) 或 `vsnprintf` 安全写入
- `buffer.data()` 确保内存对齐
