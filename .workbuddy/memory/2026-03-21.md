#### Shell View PoC 代码重大修正（11:04）

经过 MSDN + Raymond Chen 博客核查，发现上一版代码存在以下错误，已全部修正：

1. **`GetPoint` 不存在**：上一版调用 `IFolderView2::GetPoint(0, pidl, &pt)`，此方法不存在于 IFolderView2。正确方法是 `IFolderView::GetItemPosition(pidl, &pt)`，在 `IFolderView`（非 IFolderView2）上。
2. **桌面窗口查找方式错误**：上一版遍历 `LocationURL` 字符串猜测"desktop"，正确做法是 `IShellWindows::FindWindowSW(&vtLoc(CSIDL_DESKTOP), &vtEmpty, SWC_DESKTOP, ...)`。
3. **IShellBrowser 获取方式错误**：上一版直接 QI IShellBrowser，正确做法是通过 `IServiceProvider::QueryService(SID_STopLevelBrowser)` 获取。
4. **图标枚举方式错误**：上一版用索引逐个 `Item(i)`，正确做法是 `IFolderView::Items(SVGIO_ALLVIEW, IID_IEnumIDList)` 获取枚举器。
5. **方法2跨进程问题未处理**：上一版直接传本进程指针给 LVM_GETITEMPOSITION，必然失败。新版用 `VirtualAllocEx + ReadProcessMemory` 正确处理跨进程内存。

#### build_poc.bat 修复 cmd 括号解析错误（12:57）

**错误现象**：`\Microsoft was unexpected at this time.`

**根本原因**：v3 的 `if %ERRORLEVEL% NEQ 0 (...)` 括号块内有多行 `echo` 输出包含圆括号路径文本，例如 `C:\Program Files (x86)\...`。cmd 解析器在处理括号块时遇到 `)` 就认为块结束，导致后续文本（如 `\Microsoft`）成为孤立命令，触发语法错误。

**修复策略**：全面改用 `goto` 标签结构，彻底消除含路径文本的括号块。所有错误分支跳转到 `:NO_CL` / `:BUILD_FAIL` 标签，正常流程跳转到 `:DO_BUILD` / `:END`。括号块内不再有任何 `echo` 文本。
